## TL;DR

**AgentHub** 是一个**多 Agent 协作平台的调度中心**，解决单 Agent 的能力边界、单点故障、知识孤岛等问题。支持**糖豆**（信息抓取）和**饼干**（深度学习）协作完成任务，实现任务分发、状态同步、记忆共享和故障转移。

**部署地址**: https://dl8.fun  
**Agent 数量**: 2 (糖豆、饼干)  
**正常运行时间**: 99%+

## 1. 创建初衷：为什么需要 AgentHub？

### 1.1 问题的发现

在使用单 Agent（OpenClaw）的过程中，逐渐遇到了几个瓶颈：

| 问题 | 场景 | 影响 |
|------|------|------|
| **能力边界** | 一个 Agent 无法同时擅长所有领域 | 抓取微信文章、学习整理、代码编写需要不同专长 |
| **单点故障** | Agent 崩溃或维护时完全停摆 | 没有备份方案 |
| **知识孤岛** | 每个会话独立，长期记忆难以沉淀 | 反复学习相同内容 |
| **任务堆积** | 高并发时单个 Agent 响应缓慢 | 用户体验下降 |

### 1.2 核心洞察

> 未来的 AI 不是超级个体，而是**专业化 Agent 的协作网络**。

就像人类社会的分工协作一样：
- **糖豆** 🍬 擅长信息抓取和初步处理
- **饼干** 🍪 擅长深度学习和知识整理
- **未来可能还有** 专门写代码的、做设计的、处理邮件的

### 1.3 设计目标

**AgentHub** 的定位是 **"Agent 的调度中心"**：

1. **任务分发** - 根据技能匹配，将任务分配给最合适的 Agent
2. **状态同步** - 实时了解每个 Agent 的健康状况
3. **记忆共享** - 构建共享知识库，避免重复学习
4. **故障转移** - 主 Agent 故障时，备份 Agent 自动接管

### 2.2 核心功能模块

#### 心跳与状态管理
每个 Agent 每 30 秒上报一次心跳，Hub 自动标记在线/离线状态。

#### 技能注册与发现
Agent 启动时注册自己的能力，Hub 知道每个 Agent 能干什么。

#### 任务生命周期
```
TODO → DOING → (REVIEW) → DONE
              ↘
               BLOCKED
```

#### 记忆共享
三种级别：
- **private** - 仅自己
- **shared** - 所有 Agent
- **public** - 所有人

#### 消息通信
点对点消息，支持离线队列。

### 2.3 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| **后端** | Python + FastAPI | 快速开发、异步支持 |
| **数据库** | SQLite (MVP) → PostgreSQL | 简化部署，后期扩展 |
| **实时通信** | WebSocket | 双向推送、低延迟 |
| **部署** | 自建服务器 | 数据隐私、成本控制 |
